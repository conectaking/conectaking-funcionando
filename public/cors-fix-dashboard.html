Correção do erro “Failed to fetch” ao enviar imagens no banner

Se o dashboard em 127.0.0.1:5500 mostra “Failed to fetch” ou CORS ao trocar a imagem do banner, é porque as chamadas estão a ir para conectaking.com.br, que redireciona e o browser bloqueia.

Solução: o dashboard deve usar sempre a URL da API abaixo (sem redirect):

URL da API a usar no dashboard:
https://conectaking-api.onrender.com

O que fazer no projeto do dashboard

  1. Abrir o projeto do dashboard (onde está dashboard.html e dashboard.js) – tanto o que corre em localhost como o que está em produção em conectaking.com.br.
  2. Procurar onde está definida a base da API, por exemplo:
    • Variáveis como API_BASE, API_URL, apiUrl, baseURL
    • Ou o texto conectaking.com.br / www.conectaking.com.br nas chamadas fetch
  3. Substituir por: https://conectaking-api.onrender.com
  4. Ou incluir no HTML do dashboard (antes dos outros scripts): 
    <script src="https://conectaking-api.onrender.com/api-config.js"></script>
    e no JavaScript usar window.CONECTAKING_API_BASE como base das chamadas à API.

Produção (www.conectaking.com.br): se o dashboard em produção também dá "Failed to fetch", é porque aí as chamadas usam o mesmo domínio (/api/...) e o site em conectaking.com.br não é a API. Configure a base da API para https://conectaking-api.onrender.com também na build/publicação do dashboard.

Esta página é servida pela API. Guarde a URL acima no seu projeto do dashboard.
